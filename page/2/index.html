pandas 数据合并

  • pandas.merge 可用数据库表连接的方式把将不同DataFrame 中的行连接起来。

    • 参数：

    • left 参与合并的左侧 DataFrame

    • right 参与合并的右侧 DataFrame

    • how inner、outer、left、right其中之一。默认为 inner

    • on 用于连接的列名 left_on 左侧 DataFrame 中用作连接键的列

    • right_on 右侧 DataFrame 中用作连接键的列

    • left_index 布尔型，将左侧的行索引用作其连接键

    • right_index 同上

    • sort 根据连接键对合并后的数据进行排序，默认为 True。有时在处理大数据时，禁用该选项可获得更好的性能

    • suffixes 字符串值元组，用于追加到重叠列名的末尾，默认为（‘_x’，‘_y’）

    • copy 默认为 True，复制结果集

  • pandas.concat 可以沿着一条轴将多个对象堆叠到一起

    • axis : {0/’index’, 1/’columns’}, default 0

      0 表示沿-y轴拼接 ； 1表示沿x轴拼接

      join : {‘inner’, ‘outer’}, default ‘outer’

      How to handle indexes on other axis(es)

      join_axes : list of Index objects

      Specific indexes to use for the other n - 1 axes instead of performing inner/outer set logic

      用于其他n - 1轴的特定索引，而不是执行内部/外部设置逻辑

      ignore_index : boolean, default False

      If True, do not use the index values along the concatenation axis. The resulting axis will be labeled 0, …, n - 1. This is useful if you are concatenating objects where the concatenation axis does not have meaningful indexing information. Note the index values on the other axes are still respected in the join.

      如果为True，则不要使用沿连续轴的索引值。生成的轴将标记为0，…，n - 1.如果要连接并置轴没有有意义的索引信息的对象，这将非常有用。请注意，在连接中仍然遵循其他轴上的索引值。

      keys : sequence, default None

      If multiple levels passed, should contain tuples. Construct hierarchical index using the passed keys as the outermost level

      如果传递了多个级别，则应包含元组。使用传递的键作为最外层来构造层次索引

      levels : list of sequences, default None

      Specific levels (unique values) to use for constructing a MultiIndex. Otherwise they will be inferred from the keys

      用于构造MultiIndex的特定级别（唯一值）。否则，他们将从键中推断出来

      names : list, default None

      Names for the levels in the resulting hierarchical index

      生成的分层索引中的级别的名称

      verify_integrity : boolean, default False

      Check whether the new concatenated axis contains duplicates. This can be very expensive relative to the actual data concatenation

      检查是否新CONCAT enated轴包含重复。相对于实际数据连接，这可能非常昂贵

  • dataframe.append dataframe或series凭借相同列数的同类数据

    other : DataFrame or Series/dict-like object, or list of these

    The data to append.

    要追加的数据。

    ignore_index : boolean, default False

    If True, do not use the index labels.

    如果为True，请不要使用索引标签。

    verify_integrity : boolean, default False

    If True, raise ValueError on creating index with duplicates.

    如果为True，则在创建具有重复项的索引时引发ValueError。

1、添加ElRepo源

rpm --import https://www.elrepo.org/RPM-GPG-KEY-elrepo.org    



rpm -Uvh http://www.elrepo.org/elrepo-release-7.0-2.el7.elrepo.noarch.rpm

2、安装显卡驱动检查

yum install nvidia-detect

3、运行

nvidia-detect -v

390.48即为我想要的版本号

4、下载该版本的.run文件

http://www.nvidia.com/Download/driverResults.aspx/132530/en-us
提醒：可能出现页面加载不完全的情况，原因是浏览器拦截

5、进入安装前的准备工作

yum -y update //注意这是升级系统  
yum -y groupinstall "GNOME Desktop" "Development Tools"  
yum -y install kernel-devel

再次提醒：第一句话是升级系统酌情使用。

yum -y install epel-release  
yum -y install dkms

6、编辑grub文件

vim /etc/default/grub

在“GRUB_CMDLINE_LINUX”中添加

rd.driver.blacklist=nouveau nouveau.modeset=0

随后生成配置

grub2-mkconfig -o /boot/grub2/grub.cfg

7、创建blacklist

vim /etc/modprobe.d/blacklist.conf

添加

blacklist nouveau

8、更新配置

mv /boot/initramfs-$(uname -r).img /boot/initramfs-$(uname -r)-nouveau.img  
dracut /boot/initramfs-$(uname -r).img $(uname -r)

9、重启

reboot

10、确认禁用了nouveau

lsmod | grep nouveau

若无输出则禁用成功

11、开始安装

sh NVIDIA-Linux-x86_64-390.48.run

12、有用的参考：
超简单将Centos的yum源更换为国内的阿里云源 https://blog.csdn.net/chavo0/article/details/51939362

版本

  • cuda8.0
  • cudnn5.1
  • tensorflow 1.0.1

安装cuda

  • sudo sh ./cuda_8.0.44_linux.run
  • ctrl+c 直接跳过简介
  • 除网卡驱动选择no外，其余全部选yes 或 空格
  • 配置环境变量：
    • sudo gedit ~/.bashrc
    • export CUDA_HOME=/usr/local/cuda-8.0
    • export PATH=$PATH:/usr/local/cuda-8.0/bin
    • export LD_LIBRARY_PATH=/usr/local/cuda-8.0/lib64:$LD_LIBRARY_PATH
    • source ~/.bashrc
  • 测试：
    • cd /usr/local/cuda/samples/1_Utilities/deviceQuery
    • sudo make
    • ./deviceQuery

安装cudnn

  • 解压后分别进入include和lib64

  • 进入lib64文件夹后：sudo cp lib* /usr/local/cuda/lib64/

  • 进入include文件夹后：sudo cp cudnn.h /usr/local/cuda/include/

安装tensorflow

  • 官方网址下载速度太慢，可使用清华镜像网址：https://mirrors.tuna.tsinghua.edu.cn/help/tensorflow/
  • 选择合适tensorflow版本后复制网址并调用如下命令
  • sudo pip install –upgrade 网址.whl

测试

  • 需要测试tensorflow能正常打印字符串且能做数学运算
  • python
  • import tensorflow as tf
  • a=tf.constant(1)
  • b=tf.constant(1)
  • hello=tf.constant(‘hello tensorflow’)
  • sess=tf.Session()
  • print sess.run(a+b)
    • 3
  • print sess.run(hello)
    • hello tensorflow

脱坑

  • tensorflow can not find libcusolver.so.8.0
